BUILT FOR TEAMS

Modern hardware design involes a diverse array of skills and expertise, requiring cross-functional teams that must collobrate in order to succeed. Everyone on the team needs shared access to the latest version, as well as the full version history. They need the ability to experiment with new designs and ideas without having to worry about impacting the project. An OmniBuilds repository serves as the single source of truth for your project and allows for seamless collaboration across a team. Host the project in a secure online repository and acess it via the website or our cross-platform desktop app. Mirror the project on your desktop and synch directly as changes occur, giving everyone real time situational awareness.  



MODULAR PARTS & PROJECTS

Each repository includes all design files, a Bill of Materials (BOM), and all documentation for a given project. Each item in the BOM is a reference to another project repository on OmniBuilds. This modular structure allows you to easily resuse parts between designs and quickly build parts libraries. Any public project is automatically added to the open parts library and may be used as a part by any other user on the site. Private parts remain within your private parts library and are only visible to members of your team.

FULL REVISION CONTROL

Every change to any part, file, or document within your project is versioned and stored in the cloud any time you make an update from the web or destop app. In addition to simple versioning, OmniBuilds fully supports git-style workflows including branches, releases, diffs, and merges. Enjoy the freedom to experiment with alternate design paths by creating configs (branches) without having to worry about impacting the main project. Publish named revisions once your project reaches a design milestone. A standard naming convention is applied to configs and revisions to improve readability across projects. Visually diff design files in both 2D and 3D views and merge project configurations with the automated merge tool.  



CAD AGNOSTIC

OmniBuilds works with any type of CAD file, or any type of file for that matter. You have the ability to associate any file you choose as a design file for your project. This allows projects to contain Mechanical CAD, Electrical CAD, Parametric CAD, renderings, vector graphics, and the list goes on. These files are fully versioned and may be diffed against each other in 2D or 3D, depending on the format. Track your assemblies directly in a CAD file or manage each part as a discrete CAD file and organize your project with the OmniBuilds BOM tool. You have the freedom to organize your project as you need to, without having to worry about if it will work with your CAD package or team practices.
